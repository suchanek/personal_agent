# LightRAG Memory Knowledge Features Test Script

This document describes the comprehensive test script for the new LightRAG Memory Knowledge Features in the Personal AI Agent.

## Overview

The `test_lightrag_memory_features.py` script tests the enhanced memory capabilities including:

1. **Local Semantic Memory** - SQLite-based with SemanticMemoryManager
2. **Graph Memory** - LightRAG-based for relationship capture  
3. **Knowledge Base Integration** - LanceDB + LightRAG
4. **Memory Tool Integration** - Within the AgnoPersonalAgent
5. **Cross-System Memory Operations** - Consistency between systems

## Features Tested

### 1. LightRAG Servers Connectivity
- Tests connection to general knowledge LightRAG server (port 9621)
- Tests connection to memory LightRAG server (port 9622)
- Validates health endpoints

### 2. Agent Initialization
- Tests AgnoPersonalAgent initialization with memory features
- Validates memory components (SQLite, LanceDB)
- Checks available memory tools

### 3. Local Semantic Memory Operations
- Tests storing memories with topics
- Tests semantic search with similarity scoring
- Tests memory statistics generation
- Tests duplicate prevention

### 4. Graph Memory Operations
- Tests storing complex memories in LightRAG graph
- Tests various query modes (local, global, hybrid, mix)
- Tests graph labels retrieval
- Tests relationship capture

### 5. Agent Memory Tools Integration
- Tests memory tools through natural language queries
- Tests tool call detection and usage
- Tests agent responses to memory requests

### 6. Knowledge Base Integration
- Tests LanceDB knowledge base search
- Tests knowledge tools integration
- Tests agent knowledge queries

### 7. Cross-System Memory Consistency
- Tests storing same information in multiple systems
- Tests cross-system retrieval
- Tests tool usage patterns

## Usage

### Basic Usage

```bash
# Run with temporary database (recommended for testing)
python test_lightrag_memory_features.py

# Run with debug output
python test_lightrag_memory_features.py --debug

# Skip LightRAG server tests (if servers not running)
python test_lightrag_memory_features.py --skip-lightrag
```

### Advanced Usage

```bash
# Use production database (WARNING: creates test data)
python test_lightrag_memory_features.py --use-prod-db

# Custom user ID for testing
python test_lightrag_memory_features.py --user-id "my_test_user"

# Combine options
python test_lightrag_memory_features.py --debug --skip-lightrag --user-id "test_user"
```

### Command Line Options

- `--use-temp-db` - Use temporary database (default: True)
- `--use-prod-db` - Use production database (overrides --use-temp-db)
- `--skip-lightrag` - Skip LightRAG server tests
- `--debug` - Enable debug logging and verbose output
- `--user-id` - Test user ID to use (default: test_user_lightrag)

## Prerequisites

### Required Services

1. **Ollama Server** - Running with llama3.1:8b model
2. **LightRAG General Server** - Running on port 9621 (optional with --skip-lightrag)
3. **LightRAG Memory Server** - Running on port 9622 (optional with --skip-lightrag)

### Starting Services

```bash
# Start general LightRAG server
./restart-lightrag.sh

# Start memory LightRAG server  
./restart-lightrag-memory.sh

# Verify Ollama is running
ollama list
```

### Required Python Packages

The script requires these packages (should be installed in your environment):

- `aiohttp` - For HTTP requests to LightRAG servers
- `rich` - For beautiful console output
- `agno` - The agent framework
- Personal agent modules from `src/`

## Test Results

### Exit Codes

- `0` - All tests passed (100% success rate)
- `1` - Most tests passed (≥80% success rate)  
- `2` - Many failures (<80% success rate)
- `3` - Unexpected error during testing
- `130` - Tests interrupted by user (Ctrl+C)

- **Tool call detection** and usage information
- **Comprehensive summary table** with timing
- **Color-coded status indicators**
- **Debug information** when enabled
